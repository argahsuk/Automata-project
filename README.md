# DFA/NFA Simulator (Flex + Bison)

This project reads an automata description from a text file, builds an NFA, converts that NFA into a DFA using subset construction, and then simulates input strings on both machines.

## What this project does

- Parses automata definitions with Flex (`lexer.l`) and Bison (`parser.y`).
- Stores NFA components: states, alphabet, start state, final states, transitions.
- Converts NFA to DFA using bitmask-based subset construction.
- Simulates acceptance for test strings on both NFA and DFA.

## Input format

Use this format:

```txt
states: q0 q1 q2
alphabet: 0 1
start: q0
final: q2

q0 0 q0
q0 1 q1
q1 1 q2
```

Transition line format:

`<from_state> <symbol> <to_state>`

## Build and run

Example commands:

```bash
bison -d parser.y
flex lexer.l
gcc parser.tab.c lex.yy.c automata.c -o automata
./automata sample_input.txt 0 1 11 101
```

## Project files and what they do

### `automata.h`

Header file for shared constants and function declarations.

- Defines limits like `MAX_STATES`, `MAX_SYMBOLS`, `MAX_TRANSITIONS`, `MAX_DFA_STATES`, `MAX_NAME`.
- Declares functions used by parser and simulator:
  - Build/setup: `automata_begin_states()`, `automata_begin_alphabet()`, `automata_begin_finals()`, `parser_add_identifier()`, `set_start_state()`, `add_transition()`
  - Output: `print_nfa()`, `print_dfa()`
  - Processing: `convert_nfa_to_dfa()`, `simulate_nfa()`, `simulate_dfa()`

### `automata.c`

Core logic file. It stores automata data, converts NFA to DFA, and runs simulation.

Functions in this file:

- `nfa_state_idx(const char *name)`  
  Returns index of an NFA state name, or `-1` if not found.

- `nfa_sym_idx(const char *sym)`  
  Returns index of a symbol in the alphabet, or `-1`.

- `nfa_is_final(const char *name)`  
  Checks whether a state is a final state.

- `automata_begin_states(void)`  
  Sets parser mode so upcoming IDs are treated as state names.

- `automata_begin_alphabet(void)`  
  Sets parser mode so upcoming IDs are treated as alphabet symbols.

- `automata_begin_finals(void)`  
  Sets parser mode so upcoming IDs are treated as final states.

- `parser_add_identifier(const char *name)`  
  Adds parsed identifiers to the correct section (states/alphabet/finals), or consumes transition tokens in order `from -> symbol -> to`.

- `set_start_state(const char *name)`  
  Sets NFA start state.

- `add_transition(const char *from, const char *sym, const char *to)`  
  Adds one NFA transition.

- `print_nfa(void)`  
  Prints states, alphabet, start/final states, and all NFA transitions.

- `move_set(Mask mask, int si)`  
  Internal helper for subset construction/simulation: from a set of NFA states (`mask`) and symbol index (`si`), computes reachable NFA states.

- `find_dfa_state(Mask mask)`  
  Finds existing DFA state with the same subset mask.

- `mask_to_name(Mask mask, char *buf, int buflen)`  
  Converts a subset mask to readable DFA state name like `{q0,q1}`.

- `convert_nfa_to_dfa(void)`  
  Main subset construction algorithm. Builds DFA states/transitions from NFA.

- `print_dfa(void)`  
  Prints DFA states and transitions, marking start and final states.

- `simulate_nfa(const char *input)`  
  Simulates input on NFA using a bitset of current active states and prints Accepted/Rejected.

- `simulate_dfa(const char *input)`  
  Simulates input on constructed DFA and prints Accepted/Rejected.

### `parser.y`

Bison grammar file. It defines how input text is parsed and what actions are executed.

Main grammar parts:

- `program`: whole input (`sections` with optional `transitions`).
- `sections`: repeated declaration blocks.
- `section`: supports
  - `states: ...`
  - `alphabet: ...`
  - `start: ...`
  - `final: ...`
  - blank lines
- `id_list`: list of IDs in section lines.
- `transitions`: repeated transition lines.
- `transition`: one transition line (`ID ID ID`) or blank line.

Functions in this file:

- `yyerror(const char *s)`  
  Prints parser error message.

- `main(int argc, char *argv[])`  
  Program entry point. Opens input file, runs parser, prints NFA, converts to DFA, prints DFA, and simulates each extra CLI argument as a test string.

### `lexer.l`

Flex lexer file. It converts text into tokens for Bison.

Lexer rules:

- Recognizes keywords: `states`, `alphabet`, `start`, `final`.
- Recognizes `:` as `COLON`.
- Normalizes line endings (`\n`, `\r\n`, `\r`) into `NEWLINE`.
- Recognizes identifiers `[a-zA-Z0-9_]+` as `ID`.
- Ignores spaces/tabs.
- Returns raw character for unknown single characters.

Function in this file:

- `yywrap()`  
  Returns `1` to indicate lexer reached end-of-input.

### `sample_input.txt`

Example automata input file to test parsing, conversion, and simulation quickly.

### `parser.tab.c` and `parser.tab.h`

Auto-generated by Bison from `parser.y`.

- `parser.tab.c`: generated parser implementation.
- `parser.tab.h`: generated token/type declarations used by lexer and parser.

Do not edit these manually; regenerate them from `parser.y`.

### `lex.yy.c`

Auto-generated by Flex from `lexer.l`.  
Contains generated C code for lexical analysis.

Do not edit manually; regenerate it from `lexer.l`.
